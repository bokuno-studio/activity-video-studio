# macOS HIG 準拠監査レポート

- **対象**: ActivityVideoStudio（Swift / SwiftUI / AVFoundation, 最小 macOS 14, App ライフサイクル, 単一ウィンドウの動画オーバーレイ編集アプリ）
- **実施日**: 2026-06-23
- **コミット基準**: `ab412ad`（エクスポートプレビュー削除後）
- **手法**: 8 観点を並列サブエージェントで独立監査。各エージェントが実 SwiftUI ソースを `file:line` 単位で読み、現行 Apple Human Interface Guidelines と照合。
- **対象外**: `OverlayRenderer.swift` の動画**焼き込み**描画（動画コンテンツであり UI 規約の対象外）、`ExportPreviewView`（本監査と同時に削除）、`GPSTrackView.swift`（現在デッドコード。ライブプレビューは `OverlayView` の焼き込み画像を使用）。

---

## エグゼクティブサマリー

**総合判定: 機能はするが、macOS 標準からの乖離が複数領域で大きい。**

良い基盤はある — アイコンは SF Symbols を一貫使用（絵文字・自作グリフ無し）、配色は概ねセマンティックカラーでダークモードにほぼ追従、設定画面は `Form(.grouped)`、書き出し進捗は `ProgressView`(determinate)＋残り時間＋キャンセルで HIG 準拠。

一方で、**ネイティブ macOS アプリとして当然備えるべき骨格が複数欠けている**:

1. **アクセシビリティがゼロ** — `accessibility*` 修飾子がリポジトリ全体で 0 件。VoiceOver では全ボタンが SF Symbol 名で読み上げられ、再生状態も伝わらない。
2. **メニューバーが実質空** — `.commands {}` が無く、File/編集メニューやアプリ固有コマンドが存在しない。主要操作はアイコンのみのツールバーか隠しキーバインドのみ。
3. **エラー処理が貧弱** — `.alert` / `confirmationDialog` / `NSAlert` が 1 件も無い。全失敗が一過性の 9pt テキストに集約され、非対応ファイルのドロップは無反応。
4. **標準ナビゲーションコンテナ不使用** — `NavigationSplitView` / `.inspector` ではなく手組み HSplitView。ドキュメントモデル（`.avsproj`）があるのにウィンドウタイトル/プロキシアイコン無し。

### 観点別スコアカード

| 観点 | 評価 | 最重大の問題 |
|---|---|---|
| アクセシビリティ | 🔴 不良 | a11y 修飾子 0 件。VoiceOver/キーボード/コントラスト全般 |
| メニュー・キーボード | 🔴 不良 | メニューバーが空、ショートカット発見不可、Undo 無し |
| フィードバック・エラー | 🔴 不良 | alert 皆無、成功/エラー混在、サイレント失敗 |
| 色・ダークモード・素材 | 🟡 概ね良 | ドロップ背景の固定 near-black（ライトモード破綻） |
| コントロール・入力 | 🟡 概ね良 | 増減 UI を Stepper でなく自作、検証なし |
| ウィンドウ・ナビゲーション | 🟡 要改善 | 標準コンテナ/ドキュメントモデル/ツールバー配置 |
| タイポグラフィ・余白 | 🟡 要改善 | 固定 9–10pt フォント多用、8pt グリッド非準拠 |
| アイコン (SF Symbols) | 🟢 良好 | 是正不要 |

> **現状アクセシビリティ修飾子がリポジトリ全体で 0 件。** システム設定（Reduce Motion / Increase Contrast / Differentiate Without Color）の参照も皆無。

| High | `PreviewView.swift:310-337` | 再生コントロール（戻る/送る/再生・一時停止）にラベル無し。play/pause の状態が VoiceOver に伝わらない | `.accessibilityLabel(viewModel.isPlaying ? "一時停止" : "再生")` 等、状態反映 |
| High | `PreviewView.swift:385-397,481-491` | 同期 ±ボタンが「−」「＋」記号のみ。VoiceOver は記号を読むだけで量(0.5s)も向きも purpose も伝わらない | delta から `.accessibilityLabel("同期を0.5秒進める")` を生成 |
| Medium | `PreviewView.swift:259-268` | シークスライダーに label/value 無し。"slider, NN%" としか読まれず再生位置・時刻が不明 | `.accessibilityLabel("再生位置")` ＋ `.accessibilityValue(formatTime(...))` |
| Medium | `PreviewView.swift:310-337,481-491` | ヒット領域が極小（11–16pt グリフ、borderless、padding 無し）。44pt 目安を大きく下回る | `.frame(minWidth:28,minHeight:28)` か `.contentShape` ＋ padding |
| Medium | `PreviewView.swift:73-83` ほか | 9–11pt の `.tertiary/.secondary/.quaternary` 文字でコントラスト不足の懸念（4.5:1 未達リスク） | 情報文字は ≥11pt、`.tertiary`→`.secondary`/`.primary` |

## 推奨アクションプラン

監査結果を 2 グループに分けて進めることを推奨。

### グループ A — そのまま実装して良い「明確な勝ち筋」（既存レイアウトを変えない）
- **#50 アクセシビリティ** — ラベル付与＋ヒット領域拡大（追加のみ）
- **#52 エラー処理** — 失敗の alert 化、非対応ドロップへの応答、破棄確認（堅牢性向上）
- **#56 ダークモードのバグ** — 固定背景 1 点の是正
- **#51 メニューバー** — File/再生メニュー追加（既存に**足すだけ**で UX 破壊なし）

### グループ B — 見た目/操作が変わるためオーナー判断が要る
- **#53 標準コントロール** — 特に同期 ±ボタンは**オーナーが設計・承認済み**の UI。Stepper 化は要相談
- **#54 ウィンドウ構造** — `NavigationSplitView`/`.inspector`/`DocumentGroup` 移行は大きく、印象が変わる
- **#55 タイポgrafi** — フォント拡大は**現在のコンパクト密度**が変わる
