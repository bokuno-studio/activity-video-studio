## 詰まった箇所
- 長尺エクスポートで `autoExport` が複数回トリガーされる根本原因は未特定。
  `autoLoadDebugFiles` は `init()` 内の `Task { await autoLoadDebugFiles() }` から1回だけ呼ばれる設計だが、ログ上は3回 `Starting export` が出ている。
  `PreviewView` の `@StateObject private var viewModel = PreviewViewModel()` が `init()` を3回走らせている可能性が高い（SwiftUI の `WindowGroup` まわりの再生成、または長時間処理中の MainActor ブロックに起因する re-init）。
  短尺（数秒）ではタイミング的に再生成が起こる前に完了するため発症しない。

### 要修正・要確認の内容

1. **🔴 P0 リグレッション: 長尺 CLI エクスポートで output が 0 バイトになる**
   - 4時間前の前回 QA（コミット `464791e` + 8ファイル未コミット）では 37.9MB / 29.18s の正常出力。今回 (`86eb1bf`) で破壊。
   - 原因候補: 直近コミット `299a449 feat: エクスポートに GPSトラック描画 + UX修正 + CLI debug拡張` で PreviewViewModel / VideoExporter / OverlayRenderer を大幅変更（526行追加・271行削除）。
   - 推定メカニズム: `autoLoadDebugFiles`（と内部の `autoExport`）が複数回呼ばれ、`VideoExporter.exportSingleVideo` 145–146 行の `if fileExists { removeItem }` で完成済みファイルを削除→新セッションが未完了のままプロセス終了。
   - 即時対策案:
     - `autoExport` に「同一 outputURL に対する重複起動を 1回だけにする」ガード（in-flight フラグ）を追加
     - もしくは `autoLoadDebugFiles` 自体を1回しか走らないように `init()` 内のフラグで制御（`UserDefaults` か `static var didAutoLoad`）
     - 抜本的には `init()` ベースではなく `.onAppear` か NSApplicationDelegate の `applicationDidFinishLaunching` で1回だけ走らせるのが安全
   - GUI 経由の通常エクスポートには影響なし（auto 経路のみ）

2. **🟡 関連: 多重 export 発生時に AVAssetExportSession が並行する**
   - 同じ outputURL に対して2つのセッションが同時動作している痕跡（progress の振動）。長期的にはメモリ・GPU リソースを食い潰す可能性。`VideoExporter` インスタンス自体は `autoExport` 内で都度 new されるが、複数の autoExport 呼び出しが重なるとセッション衝突する。
